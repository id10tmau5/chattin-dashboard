# Changelog

All notable changes to this project are documented here.

## [1.0.1] — 2026-06-17

### Fixed
- **Loading hang on mobile (critical)** — The original `index.html` loaded Babel standalone (~4 MB library) and used it to transform 85 KB of JSX in the browser at page load time. On mobile devices this caused a 15–30+ second hang on the loading screen before anything rendered. Additionally, `libsodium-wrappers` was loaded via a CDN URL pointing to a CommonJS/Node.js build, which threw `ReferenceError: module is not defined` in the browser and blocked the page from rendering entirely.
- **Architecture pivoted to pre-compiled build:** JSX is now compiled to plain JavaScript by `build.yml` (server-side, using Babel in GitHub Actions) before being served. The browser receives ready-to-execute JS with no transform step. Page load time reduced from 15–30+ seconds to ~1–2 seconds on mobile.
- **libsodium removed from browser context** — The in-browser GitHub Secret encryption feature (which required libsodium) has been replaced with a direct link to GitHub’s repository secrets settings page. Simpler, more reliable, zero browser dependencies.

### Changed
- `index.html` reduced from ~85 KB to 1.7 KB. Now a minimal loader only — loads React from CDN, loads the compiled dashboard JS and mugshot data, mounts the app. No Babel, no libsodium.
- `chattin-case-dashboard.js` added as a new compiled output file (99 KB). Auto-generated by `build.yml`, never hand-edited.
- Edit Anthropic API Key in ⚙ Configure now opens GitHub repository secrets settings in a new tab rather than attempting in-browser encryption.

### Added
- `build.yml` GitHub Actions workflow — triggers automatically on push to `chattin-case-dashboard.jsx`, compiles JSX to JS, commits result. Enables a clean edit-push-done workflow with no manual compilation step.
